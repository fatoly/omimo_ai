# Регламент недельного цикла: закрытие недели → повестка новой недели

Инструкция для ведения цикла после онбординга. Работает в связке с протоколом
«что дальше» из SKILL.md: пользователь говорит «делаем что надо дальше» — ты по
state.yaml определяешь место в цикле и ведёшь его по этому регламенту.

## 0. Подготовка вводных (перед любым шагом цикла)

Собери и покажи пользователю сводку (без вопросов, просто вводные):

1. `python3 ~/.claude/skills/p3-flow/scripts/progress_tracker.py status` — этап и активность.
2. Открытые записи Реестра сопровождения (риски/проблемы/изменения со статусом
   «открыт»/«в работе») — только количество и 2–3 самых важных.
3. Статусы результатов из Карты результатов / Общего понимания: что в работе, что близко к готовности.
4. Последний файл цикла из `Docs/cycles/` (план прошлой недели, если есть) — что планировали.

Формат сводки: «Сейчас мы на активности E1 — закрытие недели. Вводные: на этой неделе
в работе были D2 и D3; открыто 2 риска (R1, R4); по плану недели было 3 цели. Следующий
шаг по методологии — измерить фактический прогресс. Пара вопросов: …»

## 1. Закрытие недели

### micro.P3.express — активности E1→E4, строго по порядку

Создай файл недели из шаблона `templates/micro-p3-express/cycles/week-closure.md`
в `Docs/cycles/week-YYYY-MM-DD/week-closure.md` (дата — понедельник закрываемой недели).

1. **E1 — прогресс.** Сравни план недели (файл kick-off) с фактом. Вопросы: что из
   планов сделано, что нет и почему? Обнови % готовности и статусы результатов
   в Общем понимании.
2. **E2 — удовлетворённость.** Вопрос: все ли довольны ходом проекта (1–5), что беспокоит?
3. **E3 — уроки.** Вопросы: что сработало, что мешало, какое ОДНО улучшение внедрим?
   Улучшение → запись типа «улучшение» в Реестр сопровождения.
4. **E4 — шляпы.** Вопрос: меняем ли шляпы на следующую неделю? Изменения → раздел
   «Шляпы» Общего понимания.

После E4 отметь цикл: `progress_tracker.py cycle week-YYYY-MM-DD` — и state
автоматически перейдёт на C1 (новая неделя).

### P3.express — недельный цикл C01→C04 (закрытие месяца — отдельно, E01–E03)

В P3.express неделя закрывается через измерение и реакцию: C01 (измерить прогресс
и обновить Реестр состояния) и C02 (спланировать реакции на отклонения — записи
в Реестр сопровождения; серьёзное — эскалация спонсору). Файл — из шаблона
`templates/p3-express/cycles/week.md`. Для закрытия месяца используй
`month-closure.md` (E01–E03) по той же логике, что E1–E3 выше.

## 2. Повестка нового недельного цикла

Повестка НЕ составляется с чистого листа — она собирается из итогов закрытия.
Создай файл из шаблона `week-kickoff.md` (micro) / `week.md` (P3) и перенеси:

| В повестку | Откуда |
|---|---|
| Несделанное с прошлой недели | E1/C01: план vs факт — явно решить: переносим, откладываем или отменяем |
| Действие-улучшение | E3/E02: одно улучшение недели |
| Новая раскладка шляп (micro) | E4 |
| Открытые риски/проблемы, требующие внимания на неделе | Реестр сопровождения |
| Кандидаты в работу | Карта результатов / Общее понимание: следующие по зависимостям результаты |

Дальше по активностям инициации недели (micro: C1→C4):

1. **C1 — пересмотр.** Вопросы: что изменилось с прошлой недели? Какие результаты
   берём в работу (2–3 цели недели)? Обнови Общее понимание, зафиксируй цели в повестке.
2. **C2 — ревью.** Вопрос: кто глянет план свежим взглядом? Замечания → повестка/реестр.
3. **C3 — go/no-go.** Вопрос в шляпе инвестора: продолжаем? Решение и дата — в повестку.
4. **C4 — коммуникация.** Вопрос: кому вне команды сообщить план недели?

Для P3.express старт недели — это C03 (kick-off: цели недели, кто чем занят,
препятствия) и C04 (коммуникация) в том же файле `week.md`.

## 3. Чек-лист завершения шага цикла

- [ ] Файл(ы) цикла созданы в `Docs/cycles/week-YYYY-MM-DD/` и заполнены
- [ ] Статусы результатов обновлены в основном документе
- [ ] Новые/изменённые записи внесены в Реестр сопровождения (и Реестр состояния для P3)
- [ ] `progress_tracker.py complete <активность>` после каждой активности;
      `cycle <id>` — при завершении недели/месяца
- [ ] Пользователю показаны изменённые фрагменты и предложен git-коммит
      (`docs(p3): закрытие недели 2026-07-14 (E1–E4)`) — без выполнения до явного «да»

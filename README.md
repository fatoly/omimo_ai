# omimo_ai — p3-flow

**p3-flow** — skill для [Claude Code](https://claude.com/claude-code), который проводит вас через постановку и ведение проекта по минималистичным методологиям семейства [OMIMO](https://omimo.org): **P3.express** и **micro.P3.express**.

Skill работает как живой проводник: задаёт наводящие вопросы на русском языке, по вашим ответам шаг за шагом собирает управляющие документы проекта (markdown в вашем git-репозитории), генерирует диаграммы и ведёт вас по недельным/месячным циклам методологии.

## Что умеет

- 🎓 **Обучающий онбординг** — проверяет, знакомы ли вы с OMIMO и принципами [NUPP](https://omimo.org/en/modules/nupp/manual/v1/); если нет — проводит мини-лекцию с наглядными примерами будущих артефактов.
- 🧭 **Выбор методологии** — по вопросам о размере команды, внешних сторонах и отчётности рекомендует P3.express или micro.P3.express и объясняет почему.
- 📝 **Пошаговое формирование документов** — строго по активностям методологии (A01→A10), один вопрос за раз, с показом каждого изменённого фрагмента и подтверждением перед следующим шагом.
- 🔄 **Ведение циклов** — после инициации сопровождает месячные/недельные/дневные циклы: измерение прогресса, реестр сопровождения, go/no-go, ретроспективы, «шляпы» в micro-версии.
- 🗺 **Диаграммы** — карта результатов и схема цикла генерируются через skill [Agents365-ai/drawio-skill](https://github.com/Agents365-ai/drawio-skill) (`.drawio` + PNG/SVG); без него — деградация до текстового представления.
- 💾 **Возобновление работы** — прогресс хранится в `Docs/.p3flow/state.yaml`; можно закрыть сессию и продолжить через неделю с того же места.

## Поддерживаемые методологии

| Методология | Для кого | Статус |
|---|---|---|
| [P3.express](https://omimo.org/en/modules/p3.express/manual/v2/) — 33 активности, 4 документа | команды, внешние заказчики, месячная отчётность | ✅ |
| [micro.P3.express](https://omimo.org/en/modules/micro.p3.express/manual/v1/) — 27 активностей, механика «шляп» | команды 1–7 человек | ✅ |
| [P1.express](https://omimo.org/en/modules/p1.express/) — личный тайм-менеджмент | один человек | 🔜 запланирован |

## Установка

Требуется установленный [Claude Code](https://claude.com/claude-code) и Python 3 (для вспомогательных скриптов, без внешних зависимостей).

```bash
git clone git@github.com:fatoly/omimo_ai.git
```

**Вариант А — для всех проектов** (личный skill):

```bash
cp -R omimo_ai/p3-flow ~/.claude/skills/p3-flow
```

**Вариант Б — только для одного проекта:**

```bash
cp -R omimo_ai/p3-flow <ваш-проект>/.claude/skills/p3-flow
```

Проверка: запустите `claude` и спросите «заведи проект по P3.express» — skill должен активироваться.

Опционально установите [Agents365-ai/drawio-skill](https://github.com/Agents365-ai/drawio-skill) — тогда карта результатов будет генерироваться как диаграмма, а не текстом.

## Использование

Откройте Claude Code **в git-репозитории вашего проекта** и скажите, например:

- «Заведи проект по P3.express» / «Давай оформим документацию проекта по OMIMO»
- «Помоги выбрать методологию для проекта на 3 человека»
- «Продолжим» — в репозитории с уже начатым проектом (skill сам найдёт `Docs/.p3flow/state.yaml` и восстановит контекст)

Skill создаст в репозитории каталог `Docs/`:

```
Docs/
├── .p3flow/state.yaml            # прогресс: методология, текущая активность
├── 01-project-description/       # Описание проекта
├── 02-deliverables-map/          # Карта результатов (+ diagrams/)
├── 03-follow-up-register/        # Реестр сопровождения: риски/проблемы/изменения
├── 04-health-register/           # Реестр состояния: здоровье проекта по циклам
└── cycles/                       # материалы месячных/недельных циклов
```

(для micro.P3.express — упрощённая структура: «Общее понимание проекта» + Реестр сопровождения)

Git-коммиты skill только **предлагает** — и никогда не выполняет commit/push без вашего явного подтверждения.
